# 设计改版说明（黄色渐变主题）

> 这是一份**纯前端设计改动**，未改动任何后端接口（`src/app/api/`、`src/lib/db`、`src/lib/auth` 等一律未动）。
> 配套补丁文件：`设计改版.patch`，应用方式见文末。

---

## 一、这次改了什么（概览）

把「设计工作台」页面及各内容页的视觉风格，从原来的**金色 / 酒红**主题，改造成 **HostMe 风格的黄色系**：

- 大标题深黄→浅黄渐变
- 步骤序号去掉圆圈，改成黄色渐变数字
- 模块布局放宽（不再拥挤），并排卡片改为上下堆叠
- 卡片边框建立层次：浅灰实线 / 浅灰虚线 / 淡金锚点 / 右栏深浅黄交替
- **首页完全不受影响**，共享的导航栏、背景也保持原样

共改动 **10 个文件**，无一涉及后端。

---

## 二、核心机制：`.brand-yc` 作用域（务必先理解这个）

为了做到「其它页面都变黄、但首页和共享组件保持原样」，用了 **CSS 变量作用域**方案，而不是逐页硬改颜色。

在 `globals.css` 里定义了一个作用域 class：

```css
.brand-yc {
  --accent-gold: #E9C765;   /* 强调单色：明黄 */
  --gradient-brand: linear-gradient(100deg, #C99A2E 0%, #E9C862 45%, #F6E7A6 100%);
  --border-gold: rgba(233, 199, 101, 0.3);
  --glow-gold: rgba(233, 199, 101, 0.18);
  --border: rgba(255, 255, 255, 0.1);        /* 默认卡片边框：低调浅灰 */
  --border-glow: rgba(233, 199, 101, 0.55);
}
/* 右栏编辑区细边框（颜色由各卡内联指定，深黄/浅黄交替） */
.brand-yc .edit-col .rounded-xl { border-width: 1px; }
```

**用法**：给某个页面的**最外层容器**加上 `brand-yc` 这个 class，该页所有用到 `var(--accent-gold)` / `var(--gradient-brand)` / `var(--border)` 的地方就自动变成黄色主题。

```tsx
// 例：其它页面根容器
<div className="brand-yc max-w-5xl mx-auto px-6 py-8"> ... </div>
```

> **为什么首页不变**：首页根容器**没有**加 `brand-yc`，所以它继续用 `:root` 里的原始金色变量。
> **为什么导航/背景不变**：它们在 `layout.tsx` 里、在页面容器之外，作用域管不到它们。

## 四、逐文件改动清单

### 1. `src/app/globals.css`（核心）
- 新增 `--gradient-brand` / `--brand-cyan` 等变量到 `:root`（默认金色，供首页/共享组件用）
- 新增 `.brand-yc` 作用域（见上）
- 新增 `.grad-text` 工具类（黄色渐变文字）
- 新增 `.gradient-flow` 动画类（标题渐变缓慢流动）
- 新增 `.brand-yc .edit-col .rounded-xl` 右栏细边框规则

### 2. `src/app/design/page.tsx`（改动最大）
- 顶部新增 ✦ 胶囊标签 + 大标题（深黄→浅黄渐变、流动动画）+ 副标题
- 模式切换 → 胶囊分段开关（选中态黄渐变）
- 系列 Tab → 圆润胶囊
- **步骤序号：去掉圆圈背景，改成黄色渐变大号数字**（`gradientText`）
- 布局放宽：外层 `py-10`、左右栏 `gap-10`、卡片间距 `mb-6`
- 引导模式并排的成对卡片 → 改为上下单列（`grid-cols-1`）
- 左栏卡片差异化：双手照片=淡金锚点、手型分析/拍摄指南=浅灰虚线
- **右栏（`edit-col`）卡片：深黄 `#C99A2E` / 浅黄 `#F6E7A6` 交替细边框**
- 右栏输入框字号 `text-base`、内边距加大，边框加粗突出为视觉中心
- 组合预览区=淡金锚点边框
- 顶部本地常量 `GRAD` / `gradientText` / `CYAN` 定义了本页复用的黄色渐变

### 3. `src/components/Button.tsx`
- `gold` variant 的背景从写死的金渐变改为 `var(--gradient-brand)`
- 效果：在 `brand-yc` 作用域内按钮自动变黄青渐变；作用域外（首页）仍是原金色

### 4~10. 其余 7 个页面（改动很小）
`community` / `compare` / `profile` / `tips` / `login` / `register` / `manage` 的 `page.tsx`：
- **只在根容器 className 前加了 `brand-yc `**
- 部分页面顺手把过窄的 `px-4 py-4` 放宽为 `px-6 py-8`
- **没有改任何逻辑 / 数据 / 接口调用**

## 五、如何应用补丁

在你的项目目录里执行：

```bash
# 1) 先检查能否干净应用
git apply --check 设计改版.patch

# 2a) 没报错 → 直接应用
git apply 设计改版.patch

# 2b) 若和你的改动有冲突 → 用三方合并（能自动合掉大部分）
git apply --3way 设计改版.patch

# 或者用 git am，能连同提交说明一起并入
git am < 设计改版.patch
```

### 冲突会很少，原因：
- 你的后端改动主要在 `src/app/api/` 和 `src/lib/`，**这次设计补丁完全没碰这些**
- 即使同一个 `page.tsx` 双方都动过，设计改动大多是「加 `brand-yc` class」「改颜色值」，和后端逻辑通常不在同几行

### 冲突时怎么办：
`git apply --3way` 会把冲突标在文件里（`<<<<<<<` / `>>>>>>>`），手动保留双方需要的部分即可。重点保证：
1. 各页面根容器**保留 `brand-yc`** class（否则黄色主题不生效）
2. `globals.css` 的 `.brand-yc` 作用域和工具类**要完整并入**
